# Lesson 05 - Introduction to Grasshopper

## Topic
Understanding Grasshopper and its User Interface

## What I Learned
- Learned that Grasshopper is a visual programming tool for Rhino.
- Understood that Grasshopper is about designing a **system or set of rules**, while the geometry or shape is the **output of that system**.
- Explored the Grasshopper user interface and canvas.
- Learned how components (nodes) can be connected together to create a workflow.
- Understood that data flows between components through their inputs and outputs.
- Learned how changes to inputs can automatically update the resulting geometry.
- Used the Profiler to see how long individual components take to calculate, which can help identify components that slow down a Grasshopper definition.

## Notes
This lesson introduced the basic concept behind Grasshopper. Instead of modelling a shape directly, Grasshopper allows me to design a system that generates and controls the geometry. By changing the inputs or rules of the system, the resulting geometry can update automatically.
